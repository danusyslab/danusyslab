    진행 중

1.0.0 버전 / 2023년 4월 11일 업로드

버튼은 작업을 유발하는 데 사용되는 가장 기본적인 컴포넌트입니다. 사용자는 버튼을 클릭 또는 탭할 수 있습니다.
대화상자, 모달 창, 입력 양식, 카드, 도구 모음 등 UI 전반에 걸쳐서 고루 사용됩니다.

  • 디자인 파일 바로가기
  • SCSS 파일 다운로드

구성 요소

버튼은 버튼 레이블과 버튼 아이콘, 그것들을 감싸는 컨테이너로 구성됩니다.

1. 컨테이너: 버튼 레이블과 아이콘을 포함하는, 버튼의 영역을 나타냅니다.
2. 버튼 레이블: 버튼의 동작에 대한 정보를 나타냅니다.
3. 버튼 아이콘: 버튼의 동작에 대한 정보를 나타내는 아이콘입니다.

유형

1. 크기

버튼은 그 크기에 따라 ‘small’, ‘medium’, ‘large’의 3가지 유형으로 나눌 수 있습니다.

2. 색상

버튼은 그 기능(긍정/일반/부정)에 따라 ‘primary’, ‘gray’, ‘red’의 3가지 색상 유형으로 나뉩니다.

3. 형태 및 구성

버튼은 그 형태나 구성에 따라 ‘솔리드/아웃라인/고스트’ * ‘아이콘+레이블/레이블/아이콘’의 9가지 유형으로 나눌 수 있습니다.

사용 가이드

1. 일반

- 솔리드&프라이머리 컬러 버튼은 한 화면에 하나만 있어야 합니다.
- 부정, 일반, 긍정 버튼이 함께 사용되는 경우에 부정 버튼은 좌측에, 일반 버튼은 중앙에, 긍정 버튼은 우측에 배치합니다.
- 일반 버튼 여러 개가 함께 사용되는 경우에 그 순서는 논리적(자주 사용되는 버튼 순)으로 설정하여 우선 순위가 높은 버튼을 우측에 배치합니다.
- 버튼 색상 또한 미리 설정한 규칙에 따라 결정합니다.(예. 긴급한 정보나 상태는 붉은색 계열 사용)
- 다른 곳으로 이동하는 데 버튼을 사용하지 않습니다. 링크를 사용합니다. ‘뒤로 가기’ 또는 ‘다음’ 버튼은 예외적으로 허용될 수 있습니다.

2. 레이블

- 항상 명확하고 간결한 레이블을 사용하십시오.
- 레이블은 단어나 구를 사용하며, 세 단어 이하(6글자 이하)로 가능한 짧게 작성합니다.
- 레이블에 긍정적이고 적극적인 표현을 사용합니다.

  • 솔리드&프라이머리 버튼은 한 화면에 하나만 존재

  • 동시에 2개 이상 존재 불가

  • 부정 버튼 좌측, 일반 버튼 중앙, 긍정 버튼 우측 배치

  • 자유로운 정렬 불가능

버튼 사이즈

버튼은 가장 작은 사이즈인 sm, 중간 사이즈인 md, 가장 큰 사이즈인 lg를 제공합니다.

간격 가이드

입력 버튼은 18px 사이즈를 가지며 입력 버튼과 레이블 사이 간격은 8px입니다.
라디오 버튼은 상하, 좌우 2px의 패딩 값을 가집니다.

버튼 상태 구성

기본 상태인 Default, 마우스를 올린 상태인 Hover, 마우스를 클릭한 상태인 Active, 탭키로 클릭한 상태인 Focus, 비활성화 상태인 Disabled 상태를 지원합니다.
