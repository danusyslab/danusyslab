Introduction

  • 효율적으로 운용 가능한
    디자인 업무 인력

    미리 디자인 시스템에 정의된 디자인 가이드에 따라 플랫폼을 디자인하면 ‘디자인 업무 시간’을 현저히 줄일 수 있습니다. 꼭 ‘디자이너’가 아니어도 누구나 ‘피그마’에 등록된 디자인 시스템에 따라 디자인을 할 수 있게 됩니다. ‘기획자의 와이어프레임 제작 시간’과 ‘디자이너의 디자인 시간’을 결합하여 쉽고 빠르게 플랫폼 디자인을 제작할 수 있습니다. 또한, ‘기획자’와 ‘디자이너’ 사이의 간극을 현저히 줄일 수 있으므로 협업 부서 간의 ‘미스 커뮤니케이션’을 없앨 수 있습니다.

  • 웹 접근성에 대응하는
    UI 디자인

    다누시스 디자인 시스템은 ‘웹 접근성 기준(WCAG)’에 적합합니다. 따라서 디자인 시스템에 맞춰 디자인을 하게 되면 자연스럽게 웹 접근성을 충족할 수 있습니다.

  • 일정 수준 이상의
    UI 디자인 퀄리티 유지

    디자인 시스템 가이드에 맞춰 플랫폼을 디자인하게 된다면, UI 디자인에 한하여 일정 수준 이상의 디자인 퀄리티를 유지할 수 있고 ‘디자인 일관성’을 통해 고객에게 ‘다누시스만의 공통된 디자인 이미지’를 전달할 수 있을 것입니다.

효율적으로 운용 가능한 디자인 업무 인력

미리 디자인 시스템에 정의된 디자인 가이드에 따라 플랫폼을 디자인하면 ‘디자인 업무 시간’을 현저히 줄일 수 있습니다. 꼭 ‘디자이너’가 아니어도 누구나 ‘피그마’에 등록된 디자인 시스템에 따라 디자인을 할 수 있게 됩니다. ‘기획자의 와이어프레임 제작 시간’과 ‘디자이너의 디자인 시간’을 결합하여 쉽고 빠르게 플랫폼 디자인을 제작할 수 있습니다. 또한, ‘기획자’와 ‘디자이너’ 사이의 간극을 현저히 줄일 수 있으므로 협업 부서 간의 ‘미스 커뮤니케이션’을 없앨 수 있습니다.

웹 접근성에 대응하는 UI 디자인

다누시스 디자인 시스템은 ‘웹 접근성 기준(WCAG)’에 적합합니다. 따라서 디자인 시스템에 맞춰 디자인을 하게 되면 자연스럽게 웹 접근성을 충족할 수 있습니다.

일정 수준 이상의 UI 디자인 퀄리티 유지

디자인 시스템 가이드에 맞춰 플랫폼을 디자인하게 된다면, UI 디자인에 한하여 일정 수준 이상의 디자인 퀄리티를 유지할 수 있고 ‘디자인 일관성’을 통해 고객에게 ‘다누시스만의 공통된 디자인 이미지’를 전달할 수 있을 것입니다.

  • As-is

    기존의 UI 디자인은 디자이너가 프로젝트에 따라, 프로젝트 마다 새로운 디자인을 제작해야했습니다. 그렇다보니 UI 디자인 사이의 ‘디자인 일관성’을 유지하기 힘들고 디자인 작업 시간 또한 많이 소요되었습니다.

  • To-be

    디자인 시스템을 활용하여 디자인 시안을 빠르게 제작할 수 있으며 UI 디자인 영역이 디자이너만 작업 가능한 영역이 아닌 ‘누구나 작업 가능한’ 영역 으로 확장되었습니다. 또한 프로젝트는 달라도 ‘다누시스만의 디자인 일관성’을 유지할 수 있습니다.
